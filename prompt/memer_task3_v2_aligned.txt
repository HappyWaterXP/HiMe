You will be given:
- Memory keyframes: older composite images from the past.
- Recent frames: up to K composite images, indexed from 1..K.

Task:
{global_instruction}

# Core Rules

- Recent frames are the current world state.
- Use recent frames as the main evidence for:
  - what is true now
  - what the next action should be
- Memory keyframes are historical context from the past.
- Never let memory keyframes override what is visible in recent frames.
- Use memory keyframes only as auxiliary context for object identity, history, and user preference.
- If recent frames and memory keyframes disagree, trust recent frames.
- Time matters:
  - memory keyframes are older
  - recent frames are newer and closer to the present
- Use memory mainly when recent frames do not clearly show an object, location, or user preference.

# Image Reading

- Each frame is a composite image:
  - left half: wrist / waist view
  - right half: main / head view
- Use both views together.
- Recent frames preserve temporal order.
- Earlier recent frames show earlier moments. Later recent frames are closer to the current final state.
- The latest recent frame has the highest priority for judging what is true now.
- Middle recent frames may show intermediate motion states, not the final result.
- Use the recent sequence to understand motion, but use the latest recent frame(s) to decide the current state.

# User Preference

- If the user instruction explicitly states a preference, constraint, or future intent, keep it in mind when choosing the next action.
- Do not ignore an explicit user preference if it affects the next action choice.

# Action Format Rules

You MUST output exactly ONE action string in ONE of the following formats:

Format A (inspection):
- `inspect <target>`
- Use inspection only when information required for a correct pick-and-place action is missing or uncertain.
- The target should be a concrete location or container.

Format B (pick and place):
- `pick up <object> <source_location> and place it <target_location>`
- In task3-style tasks, object names should use only:
  - `the toy croissant`
  - `the toy mushroom`
  - `the toy bread`
- In task3-style tasks, location phrases should use only:
  - `on the left plate`
  - `on the right plate`
  - `on the table`
  - `in the box`
- Do not use `from`, `to`, `move`, or alternate wording in the action string.
- One pick-and-place action moves only one object.

# Keyframe Selection Rules

- Output `keyframe_positions` as indices chosen only from the recent frames.
- Do not choose memory keyframes in `keyframe_positions`.
- Use `keyframe_positions` to nominate recent frames that are useful evidence for future memory/keyframe maintenance.
- `keyframe_positions` must be:
  - unique
  - sorted
  - each integer must be in `[1, K]`

# Output Format

Return JSON only with exactly:

{{ "action": string, "keyframe_positions": number[] }}

No extra keys.
No explanations.
No markdown.
